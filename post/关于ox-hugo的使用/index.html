关于文件头

“#+hugo_base_dir: ~/blog” : hugo的根目录

“#+hugo_section: post” ： section的地址，我选用的这个博客主题要求我section地址为post,这样 文件就会输出到 ~/blog/content/post/

“#+hugo_auto_set_lastmod: t” ox-hugo会自动更新博客最新修改时间

“#+title : name” :设置整个文件导出后的名称

关于org 分支的properties 的设置

“:EXPORT_FILE_NAME:” name : 分支导出后的文件名为 name.md

默认按键

C-c C-e H H

根据光标位置导出一个分支，或是导出整个文件

C-c C-e H A

导出所有分支，或是导出整个文件

标记

将分支设为todo，ox-hugo导出时会将其标记为草稿

将分支设为done,ox-hugo导出时会利用done 记录的时间作为日期

对分支进行schedule 的设置，会设置为hugo 中的 publishDate

图像

在hugo 中调用本地图片如 ~/hugo/static/images/foo.png 只需要用 /images/foo.png 这样的路径, 所以所有图片资源都放static 文件夹就行

格式

代码块

为了hugo 的正确解析，需要将config.toml 中 pygmentsCodeFences 设为 true

例子

org 下格式：

" #+begin_src emacs-lisp -n :hl_lines 2

;; this will export with line number 1 (default)

(message “yes”)

“#+end_src 输出结果：

1
2

;; this will export with line number 1 (default)
(message  "%s" "This is line 2")

解析

-n 使得导出时是带有行号的 +n 会自动沿续之前的行号 :hl_lines可以设置代码行的高亮

标签与类别

  1. :tag1:单个分支的tag #+filetags整个文件中分支都有的tag
  2. :@cat1:设置单个分支的类别
  3. 注意 abc_def 会转化为 tag “abc-def”, and abc__def 会转化为 tag “abc def”.
